How "live" works

The GitHub Actions workflow runs every weekday at 22:00 UTC (6pm ET, after US market close). Each run pulls fresh prices from Yahoo Finance, regenerates all signals, runs a 1500-sample backtest with 5-fold cross-validation, updates pattern weights, and commits the new JSON to this page. GitHub Pages auto-deploys the updated site within ~1 minute. To force an immediate refresh (e.g., during market hours for after-the-close analysis), click "↻ Refresh now" in the header — it opens the Actions tab where you can press "Run workflow" to trigger a run on-demand (takes ~70 seconds). True real-time (intraday) data would require a separate backend service; daily-bar technical analysis doesn't gain much from intraday updates since the bar doesn't close until 4pm ET.

Backtest details

Each run takes 1500 random (ticker, past-date) samples and predicts forward using only data available at that date. Tickers are sampled uniformly from a ~200-name universe (large caps, ETFs, popular trading names) so the test is not biased toward any specific stock.

How it works

Every pattern uses a different historical window. The ensemble combines them so the final call is informed by short, medium, and long term context simultaneously.

Short-term context (5–14 days)

  • RSI(14) — momentum over 2 trading weeks
  • 5-day SMA for multi-timeframe alignment
  • Candlestick patterns: engulfing, hammer, shooting star
  • Volume breakouts (z-score over 20 bars)
  • ATR(14) for stop-loss distance

Medium-term context (20–50 days)

  • 20-day SMA + Bollinger Bands (squeeze over 60-bar window)
  • MACD(12/26/9) — 12 and 26 day EMAs
  • 50-day SMA — primary swing trend
  • Relative strength vs SPY over 20 days
  • Volume dry-up / dry-top divergence (20-bar lookback)

Long-term context (200+ days)

  • 200-day SMA — long-term trend
  • Golden cross / death cross (50d vs 200d)
  • Multi-timeframe stack: price > 5/20/50/200 SMA
  • Market regime: SPY vs 200-SMA + 60-bar slope

Cross-cutting

  • Per-horizon weight tuning (5d/10d/20d/60d/120d/252d separately)
  • Per-horizon methodology filtering (meta only uses methods with positive edge AT THAT horizon)
  • K-fold cross-validation (k=5) for honest out-of-sample accuracy
  • Auto-pruning: methodologies below chance at every horizon are dropped
  • Market regime tagging on every backtest sample
  • News sentiment via VADER on Yahoo Finance headlines (live only)
  • Strict no-look-ahead: cutoff dates see only past data

Not investment advice. Pattern-based predictions are probabilistic and can be wrong. Read the K-fold accuracy and methodology leaderboard before acting on a signal. Source: github.com/hunain-malik/investing-platform
