How "live" works + how to refresh manually

The workflow runs 4 times per US weekday (14:30, 17:00, 19:30, 22:00 UTC) plus on manual trigger. The 22:00 UTC run is authoritative (uses the closing bar); intraday runs include partial bars and current prices.

To run analysis right now (~3 minutes):

  1. Click this link to open the Actions page
  2. On that page, click the gray "Run workflow ▾" button (top right of the runs list)
  3. In the dropdown that appears, click the green "Run workflow" button
  4. Wait ~3 minutes, then refresh this dashboard

GitHub Pages doesn't let us run Python on click directly — but this two-click sequence is the official path. A future iteration could add a small serverless trigger (Cloudflare Workers) for one-click refresh.

How it works (patterns + windows + sources)

Short-term (5–14 days)

  • RSI(14), 5-day SMA
  • Candlestick: engulfing, hammer, shooting star, doji
  • Volume breakouts (20-bar z-score)
  • ATR(14) for stop-loss

Medium-term (20–50 days)

  • 20/50 SMA, Bollinger Bands (60-bar squeeze)
  • MACD(12/26/9)
  • Relative strength vs SPY (20d)
  • Volume dry-up / dry-top divergence

Long-term (200+ days)

  • 200-SMA, golden/death cross
  • Multi-timeframe stack (5/20/50/200)
  • Market regime (SPY vs 200-SMA + 60-bar slope)

Cross-cutting

  • Per-horizon weight tuning
  • 5-fold cross-validation for out-of-sample accuracy
  • Auto-pruning of below-chance methodologies
  • Earnings-date awareness via Yahoo Finance calendar
  • Headline sentiment (VADER) on Yahoo Finance news
  • Asymmetric bearish quorum (markets drift up; bearish calls need 3+ methodologies vs 2 for bullish)

Methodology sources

  • Murphy, "Technical Analysis of the Financial Markets" — SMA crossovers, MACD, support/resistance, trend definitions
  • Nison, "Japanese Candlestick Charting Techniques" — engulfing, hammer, shooting star, doji
  • Wilder, "New Concepts in Technical Trading Systems" (1978) — RSI(14), ATR
  • Bollinger, "Bollinger on Bollinger Bands" — band squeeze + breakout setups
  • Jegadeesh & Titman (1993) — relative strength / momentum factor
  • Stacking ensembles — standard ML practice; the meta-ensemble weights each sub-method by its k-fold out-of-sample accuracy
  • Methodology auto-pruning — drop below-chance approaches before they pollute the ensemble (standard ML hygiene)

Not investment advice. K-fold CV reveals technical-analysis-only signals have near-coin-flip accuracy out-of-sample. Use this for entry timing on positions you've already decided on for other reasons, not as a standalone prediction engine. Source: github.com/hunain-malik/investing-platform
